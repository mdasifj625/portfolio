# Building TarbiyahOS with Next.js and Supabase

Building TarbiyahOS was an engineering challenge focusing on creating a multi-tenant SaaS application that performs reliably under heavy load. The decision to use **Next.js** for the frontend and **Supabase** for the backend stemmed from the need for high velocity and solid stability.

## Architecture Decisions

### 1. The Frontend: Next.js App Router
Choosing the Next.js App Router allowed us to heavily utilize Server Components, drastically reducing the client-side JavaScript bundle. This led to faster Time To Interactive (TTI) and better SEO for our public-facing pages.

### 2. The Backend: Supabase (PostgreSQL)
We bypassed traditional ORMs and REST APIs in favor of Supabase's direct Postgres connection and built-in Row Level Security (RLS). This enabled us to:
- Enforce tenant isolation directly at the database level.
- Stream real-time updates to connected clients using PostgreSQL triggers.

### 3. Analytics and Monitoring
- **PostHog**: Integrated for feature flags and product analytics.
- **Sentry**: Critical for capturing edge-case errors in our distributed systems.

## Challenges & Learnings
The primary challenge was managing complex state across the application without relying heavily on global state managers like Redux. By relying on React Server Components and React Query for client-side caching, we kept the architecture mental model simple and predictable.
